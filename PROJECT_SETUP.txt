================================================================================
                           4. RUNNING THE PROJECT
================================================================================

----------------------
OPTION A: Using Docker (Recommended)
----------------------

1. Install Docker Desktop:
   https://www.docker.com/products/docker-desktop/

2. Copy environment file:
   cp .env.example .env

3. Update .env with your configuration

4. Start all services:
   docker-compose up -d

5. Access the application:
   - Frontend: http://localhost:8015
   - Backend API: http://localhost:8016
   - API Docs: http://localhost:8016/api/docs/

6. Stop services:
   docker-compose down


----------------------
OPTION B: Running Locally
----------------------

PREREQUISITES:
  
  macOS:
    brew install gdal postgresql@15 postgis
    brew services start postgresql@15
  
  Ubuntu/Debian:
    sudo apt install gdal-bin libgdal-dev postgresql-15 postgresql-15-postgis-3
    sudo systemctl start postgresql

STEP 1: Set up PostgreSQL Database
-----------------------------------
  # Create database
  createdb globalrouter
  
  # Enable PostGIS extension
  psql -d globalrouter -c "CREATE EXTENSION IF NOT EXISTS postgis;"


STEP 2: Install Backend Dependencies
------------------------------------
  cd backend/server
  
  # Create virtual environment (recommended)
  python3 -m venv venv
  source venv/bin/activate  # Linux/macOS
  # or: venv\Scripts\activate  # Windows
  
  # Install dependencies
  pip install -r requirements.txt
  
  # Copy environment file
  cp sample.env .env
  # Edit .env with your database credentials


STEP 3: Run Django Migrations
-----------------------------
  cd backend/server
  python3 manage.py migrate
  
  # Create superuser
  python3 manage.py createsuperuser


STEP 4: Start Backend Server
----------------------------
  cd backend/server
  python3 manage.py runserver
  
  # Backend will run at: http://127.0.0.1:8000


STEP 5: Install Frontend Dependencies
-------------------------------------
  cd frontend
  
  # Using npm:
  npm install
  
  # Or using pnpm (recommended):
  pnpm install
  
  # Copy environment file
  cp sample.env .env


STEP 6: Start Frontend Server
-----------------------------
  cd frontend
  npm run dev
  
  # Frontend will run at: http://localhost:5173

                           5. API DOCUMENTATION
================================================================================

Global Router uses Django REST Framework with drf-yasg for API documentation.

API ENDPOINTS:
--------------
Base URL: http://localhost:8000/api/ (local) or http://localhost:8016/api/ (Docker)

Authentication:
  POST   /api/auth/login/           - User login
  POST   /api/auth/logout/          - User logout
  POST   /api/auth/register/        - User registration
  GET    /api/auth/user/            - Get current user

Adventures:
  GET    /api/adventures/           - List all adventures
  POST   /api/adventures/           - Create new adventure
  GET    /api/adventures/{id}/      - Get adventure details
  PUT    /api/adventures/{id}/      - Update adventure
  DELETE /api/adventures/{id}/      - Delete adventure

Collections (Itineraries):
  GET    /api/collections/          - List collections
  POST   /api/collections/          - Create collection
  GET    /api/collections/{id}/     - Get collection details
  PUT    /api/collections/{id}/     - Update collection
  DELETE /api/collections/{id}/     - Delete collection

World Travel:
  GET    /api/worldtravel/countries/     - List all countries
  GET    /api/worldtravel/regions/       - List regions
  GET    /api/worldtravel/visited/       - Get visited locations

Integrations:
  GET    /api/integrations/strava/       - Strava integration
  POST   /api/integrations/gpx/upload/   - Upload GPX file


SWAGGER DOCUMENTATION:
----------------------
  - Swagger UI: http://localhost:8000/api/docs/
  - ReDoc: http://localhost:8000/api/redoc/
  - OpenAPI Schema: http://localhost:8000/api/schema/


AUTHENTICATION:
---------------
The API uses session-based authentication with django-allauth.

For API requests, include the session cookie or use token authentication:
  Authorization: Token <your_token>

                           6. TROUBLESHOOTING
================================================================================

COMMON ISSUES:

1. "Could not find the GDAL library"
   --------------------------------
   Solution: Install GDAL
   - macOS: brew install gdal
   - Ubuntu: sudo apt install gdal-bin libgdal-dev

2. "Connection refused" for PostgreSQL
   -----------------------------------
   Solution: Ensure PostgreSQL is running
   - macOS: brew services start postgresql@15
   - Ubuntu: sudo systemctl start postgresql

3. "PostGIS extension not found"
   -----------------------------
   Solution: Install PostGIS and create extension
   - macOS: brew install postgis
   - Then: psql -d globalrouter -c "CREATE EXTENSION postgis;"

4. "Port already in use"
   ---------------------
   Solution: Change port in .env or kill existing process
   - Find process: lsof -i :5173
   - Kill process: kill -9 <PID>

5. Frontend shows API errors
   -------------------------
   Solution: Ensure backend is running and PUBLIC_SERVER_URL is correct
   - Check backend: curl http://127.0.0.1:8000/api/
   - Verify .env: PUBLIC_SERVER_URL=http://127.0.0.1:8000
